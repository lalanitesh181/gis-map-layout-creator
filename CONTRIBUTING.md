# Contributing

Thanks for your interest in contributing to Academic GIS Pro.

## Getting Started

1. Fork the repository
2. Create a feature branch
3. Make focused changes with clear commit messages
4. Test in browser (`npx --yes serve -l 5173`)
5. Open a pull request with a short description and screenshots for UI changes

## Guidelines

- Keep code readable and modular
- Avoid introducing large dependencies unless necessary
- Preserve current UI behavior unless change is intentional
- For bug fixes, include a short reproduction note in the PR

## Pull Request Checklist

- [ ] Change is scoped and documented
- [ ] No obvious console errors in browser
- [ ] UI remains usable on standard desktop screens
- [ ] Export flow (PNG/JPG/PDF) still works for impacted areas
